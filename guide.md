# ML Network Lab — Guide

This lab teaches you how to build an **automatic router health classifier** using Python and scikit-learn. You will start from scratch — no prior Machine Learning knowledge required.

By the end you will know how to generate a dataset, train a Random Forest, evaluate its performance, and make predictions on new data.

## Prerequisites

Start the lab and wait for the boot to complete (~2-3 minutes, including pip install):

```bash
qlab run ml-network-lab
```

Connect via SSH:

```bash
qlab shell ml-network-lab
```

Verify cloud-init has finished (wait for "status: done"):

```bash
cloud-init status --wait
```

Verify scikit-learn is installed:

```bash
python3 -c "import sklearn; print('scikit-learn', sklearn.__version__)"
```

## Exercise 2: Analysing the dataset

After running module 02, the dataset is at `~/data/router_dataset.csv`. Explore its structure:

```python
import pandas as pd
df = pd.read_csv("/home/labuser/data/router_dataset.csv")

# Structure
print(df.shape)          # (500, 10)
print(df.dtypes)

# First rows
print(df.head())

# Statistics
print(df.describe())

# Class distribution
print(df["status"].value_counts())
```

**Questions to explore:**
1. How many routers are in the "critical" state? Is the dataset imbalanced?
2. Which column has the highest standard deviation?
3. Is there a correlation between CPU and temperature?

```python
# CPU vs temperature correlation
print(df[["cpu_usage_pct", "temperature_celsius"]].corr())

# Mean CPU per class
print(df.groupby("status")["cpu_usage_pct"].mean())
```

## Exercise 3: Interpreting the model

After running modules 04, 05, and 06, analyse the feature importances:

```python
import pickle, numpy as np

with open("/home/labuser/data/random_forest.pkl", "rb") as f:
    model = pickle.load(f)
with open("/home/labuser/data/feature_names.pkl", "rb") as f:
    feats = pickle.load(f)

# Sorted feature importances
importances = model.feature_importances_
idx = np.argsort(importances)[::-1]

print("Feature importances:")
for i in idx:
    print(f"  {feats[i]:25s}: {importances[i]:.4f}")
```

**Questions to explore:**
1. Which feature contributes most to the model's decisions?
2. Does it match what you expected from the correlation analysis?
3. What would happen if you removed the least important feature?

### Challenge 3: GridSearchCV

Automatically find the best hyperparameters:

```python
from sklearn.model_selection import GridSearchCV

param_grid = {
    "n_estimators": [50, 100, 200],
    "max_depth": [5, 10, 15, None],
}
gs = GridSearchCV(RandomForestClassifier(random_state=42),
                  param_grid, cv=5, scoring="f1_macro", n_jobs=-1)
gs.fit(X_train, y_train)
print("Best parameters:", gs.best_params_)
print("Best F1 macro:", gs.best_score_)
```
